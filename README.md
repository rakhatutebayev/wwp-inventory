# WWP Inventory - Система учета компьютерной техники

Полнофункциональная система учета компьютерной техники с веб-интерфейсом и мобильным приложением.

## Структура проекта

- `backend/` - FastAPI backend с PostgreSQL
- `frontend/` - React веб-приложение
- `mobile/` - React Native мобильное приложение

## Требования

- Python 3.9+
- Node.js 18+
- PostgreSQL 12+
- npm или yarn

## Установка и запуск

### Backend

1. Перейдите в директорию backend:
```bash
cd backend
```

2. Создайте виртуальное окружение:
```bash
python -m venv venv
source venv/bin/activate  # На Windows: venv\Scripts\activate
```

3. Установите зависимости:
```bash
pip install -r requirements.txt
```

4. Настройте базу данных PostgreSQL и создайте файл `.env`:
```
DATABASE_URL=postgresql://user:password@localhost:5432/wwp_inventory
SECRET_KEY=your-secret-key-change-in-production
```

5. Создайте базу данных:
```bash
createdb wwp_inventory
```

6. Запустите сервер:
```bash
uvicorn app.main:app --reload
```

API будет доступен по адресу: http://localhost:8000

### Frontend

1. Перейдите в директорию frontend:
```bash
cd frontend
```

2. Установите зависимости:
```bash
npm install
```

3. Запустите dev сервер:
```bash
npm run dev
```

Веб-приложение будет доступно по адресу: http://localhost:3000

### Mobile

1. Перейдите в директорию mobile:
```bash
cd mobile
```

2. Установите зависимости:
```bash
npm install
```

3. Запустите Expo:
```bash
npm start
```

4. Отсканируйте QR-код в приложении Expo Go на вашем телефоне или запустите на эмуляторе.

## Основной функционал

### Справочники
- Типы устройств (Монитор, Принтер, Телефонный Аппарат, Коммутатор и т.д.)
- Бренды
- Модели
- Сотрудники (ФИО, телефонный номер 3 цифры)
- Склады

### Устройства
- Создание устройств с указанием типа, бренда, модели
- Уникальные серийные номера и инвентарные номера
- Отслеживание текущей локации (склад или сотрудник)

### Перемещения
- История всех перемещений устройств
- Автоматическое создание записи при перемещении
- Дата и пользователь, выполнивший перемещение

### Отчеты
- Список всех устройств с фильтрацией
- История перемещений конкретного устройства
- Отчет по локациям (что где находится)
- Экспорт в CSV

## API Endpoints

### Аутентификация
- `POST /api/auth/register` - Регистрация
- `POST /api/auth/login` - Вход
- `GET /api/auth/me` - Текущий пользователь

### Справочники
- `GET /api/device-types` - Список типов устройств
- `GET /api/brands` - Список брендов
- `GET /api/models` - Список моделей
- `GET /api/employees` - Список сотрудников
- `GET /api/warehouses` - Список складов

### Устройства
- `GET /api/devices` - Список устройств
- `POST /api/devices` - Создать устройство
- `GET /api/devices/{id}` - Получить устройство
- `PUT /api/devices/{id}` - Обновить устройство
- `DELETE /api/devices/{id}` - Удалить устройство

### Перемещения
- `GET /api/movements` - Список перемещений
- `POST /api/movements` - Создать перемещение

### Отчеты
- `GET /api/reports/devices` - Отчет по устройствам
- `GET /api/reports/devices/export` - Экспорт в CSV
- `GET /api/reports/locations` - Отчет по локациям

## Мобильное приложение

Мобильное приложение включает:
- Сканирование QR-кодов для быстрого поиска устройств
- Просмотр списка устройств
- Детальная информация об устройстве
- Создание перемещений
- Просмотр истории перемещений
- Отчеты

## Безопасность

- JWT аутентификация
- Хеширование паролей (bcrypt)
- Валидация уникальности серийных и инвентарных номеров на уровне БД
- CORS настройки
